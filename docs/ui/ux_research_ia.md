# UI: UX Research & IA

## Personas

- **Listener**: Wants fast discovery and seamless playback.
- **Artist**: Needs upload, pricing, and performance insights.
- **Curator**: Stakes on emerging artists and tracks outcomes.

## Primary Journeys

1. **Listener session**
   - Open app → discover mood/genre → start session → now playing → queue.
2. **Artist upload**
   - Upload audio → review stems → set pricing → publish.
3. **Artist analytics**
   - View plays/payouts → drill into track performance.

## Sitemap (MVP)

- **Home / Discover**
- **Now Playing**
- **Queue**
- **Search**
- **Artist Upload**
- **Artist Analytics**
- **Wallet / Budget**
- **Settings**

## Navigation Model

- **Left sidebar**: Home, Search, Library, Artist tools.
- **Top bar**: Search + profile.
- **Bottom bar**: Player (now playing, progress, controls).

## Priority Flows

- Play a track in < 2 clicks.
- Start a session in < 30 seconds.
- Upload a track in < 3 minutes.
